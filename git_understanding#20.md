1. What makes a good commit message?
Short and clear.
Explains what changed and why.
Starts with an action word like:
Add, Fix, Update, Remove, Refactor.
Avoids unclear words like “update”, “changes”, “stuff”.
Good examples:
Fix timer freeze bug in Focus Mode
Add validation for user input
Refactor login function for readability
Bad examples:
update
fix
changes

2. Why meaningful commit messages matter
Helps teammates understand your work quickly.
Makes debugging easier.
Keeps project history clean.
Useful during pull request reviews.

less

4. What I learned
Good messages save time for future debugging.
They make the project history easier to read.
They show professionalism in teamwork.
